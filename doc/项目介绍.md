# easybuild-admin 接口示例项目（demo-bms）

## 项目简介

`demo-bms` 是为 **easybuild-admin** 中后台项目提供的一个简单的单体接口示例项目，方便开发者快速体验 easybuild-admin 中后台项目的完整功能。项目本身不包含任何业务代码，所有业务功能均通过 Maven 依赖引入的方式集成。开发者可以根据自身需求选择引入不同的业务模块，快速搭建一套完整的后台管理系统。

### 技术栈

- **JDK**: 21
- **框架版本**: easybuild-admin 3.2.12
- **ORM**: MyBatis-Plus / MyBatis-Flex / Hibernate（可选）
- **缓存**: Redis（Lettuce 连接池）
- **接口文档**: Knife4j
- **日志**: Logback
- **构建工具**: Maven

## 数据库选择与配置

项目支持 **MySQL** 和 **PostgreSQL** 两种数据库，通过切换依赖和配置文件即可完成切换。

### 方式一：使用 MySQL（默认）

#### 1. pom.xml 依赖配置

确保引入 MySQL 依赖，注释掉 PostgreSQL 依赖：

```xml
<!-- mysql数据库依赖 -->
<dependency>
    <groupId>com.mcst</groupId>
    <artifactId>db-mysql</artifactId>
</dependency>

<!-- postgresql数据库依赖（注释掉） -->
<!--<dependency>
    <groupId>com.mcst</groupId>
    <artifactId>db-postgresql</artifactId>
</dependency>-->
```

#### 2. 主配置文件 application.yml

确保引入 MySQL 对应的环境配置文件：

```yaml
spring:
  config:
    import:
      - "file:config/application-dev.yml"
```

#### 3. 环境配置文件 application-dev.yml

配置 MySQL 数据源连接信息：

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/eb_demo?characterEncoding=UTF-8&useSSL=false&useInformationSchema=true&remarks=true&useUnicode=true&allowPublicKeyRetrieval=true
    username: root
    password: 123456
    type: com.zaxxer.hikari.HikariDataSource
    driver-class-name: com.mysql.cj.jdbc.Driver
```

#### 4. 初始化数据库

使用 `doc/sql/mysql_260322.sql` 脚本创建数据库和表结构。脚本中已包含初始化数据，执行后即可直接使用。

### 方式二：使用 PostgreSQL

#### 1. pom.xml 依赖配置

引入 PostgreSQL 依赖，注释掉 MySQL 依赖：

```xml
<!-- postgresql数据库依赖 -->
<dependency>
    <groupId>com.mcst</groupId>
    <artifactId>db-postgresql</artifactId>
</dependency>

<!-- mysql数据库依赖（注释掉） -->
<!--<dependency>
    <groupId>com.mcst</groupId>
    <artifactId>db-mysql</artifactId>
</dependency>-->
```

#### 2. 主配置文件 application.yml

切换为 PostgreSQL 对应的环境配置文件：

```yaml
spring:
  config:
    import:
      - "file:config/application-pgDev.yml"
```

#### 3. 环境配置文件 application-pgDev.yml

配置 PostgreSQL 数据源连接信息：

```yaml
spring:
  datasource:
    url: jdbc:postgresql://127.0.0.1:5432/eb_demo
    username: postgres
    password: 123456
    type: com.zaxxer.hikari.HikariDataSource
    driver-class-name: org.postgresql.Driver
```

#### 4. 初始化数据库

使用 `doc/sql/postgresql_260322.sql` 脚本创建数据库和表结构。脚本中已包含初始化数据，执行后即可直接使用。

## 其他配置说明

### 服务端口

在 `config/application.yml` 中配置：

```yaml
server:
  port: 5001
```

### 系统初始化（仅供新项目参考）

环境配置文件中提供了系统用户和资源自动初始化的配置项：

```yaml
easyfk:
  config:
    auth:
      init: true          # 是否初始化系统用户
    resource:
      init:
        open: true         # 是否初始化系统资源
```

> **说明**：本示例项目提供的 SQL 脚本中已包含完整的初始化数据（管理员账号、资源权限等），因此 **无需开启** 上述初始化配置，保持 `false` 即可。此配置仅作为新项目（使用空数据库）的参考，新项目首次启动时可设为 `true` 来自动初始化基础数据。

### 接口文档（Knife4j）

```yaml
knife4j:
  enable: true            # 启用接口文档增强
  production: false        # 生产环境设为 true 以屏蔽文档
```

启动后访问接口文档地址：`http://localhost:5001/doc.html`

## 启动项目

### 前置条件

1. **JDK 21** 已安装并配置环境变量
2. **Maven** 已安装
3. **数据库**（MySQL 或 PostgreSQL）已安装并运行
4. **Redis** 已安装并运行
5. 已执行对应的数据库初始化 SQL 脚本

### 启动步骤

#### 1. 克隆项目

```bash
git clone <仓库地址>
cd demo-bms
```

#### 2. 选择数据库并修改配置

参考上文「数据库选择与配置」章节，完成 `pom.xml` 依赖切换和配置文件修改。

#### 3. 编译项目

```bash
mvn clean package -DskipTests
```

#### 4. IDE 启动（开发模式）

在 IDE（如 IntelliJ IDEA）中直接运行启动类：

```
com.easybuild.admin.bms.ServerApp
```

#### 5. 命令行启动

```bash
java -jar target/demo-bms-server.jar
```

#### 6. 验证启动

- 控制台出现 Spring Boot 启动成功日志
- 访问接口文档：`http://localhost:5001/doc.html`

## ORM 框架选择

项目支持三种 ORM 框架，本示例默认使用 **MyBatis-Plus**。每个业务模块的 ORM 层依赖遵循统一的命名规则：

| ORM 框架 | 依赖后缀 | 示例（以 banner 模块为例） |
|----------|---------|--------------------------|
| MyBatis-Plus | `orm-plus` | `banner-orm-plus` |
| MyBatis-Flex | `orm-flex` | `banner-orm-flex` |
| Hibernate | `orm-hibernate` | `banner-orm-hibernate` |

### 如何切换 ORM 框架

如需将 ORM 从 MyBatis-Plus 切换为 MyBatis-Flex 或 Hibernate，需要替换 **每个业务模块** 的 ORM 依赖。

例如，将 banner 模块从 MyBatis-Plus 切换为 MyBatis-Flex：

```xml
<!-- 替换前：MyBatis-Plus -->
<dependency>
    <groupId>com.mcst</groupId>
    <artifactId>banner-orm-plus</artifactId>
</dependency>

<!-- 替换后：MyBatis-Flex -->
<dependency>
    <groupId>com.mcst</groupId>
    <artifactId>banner-orm-flex</artifactId>
</dependency>
```

> **注意**：切换 ORM 时需要将 `pom.xml` 中 **所有业务模块** 的 `orm-plus` 依赖统一替换（如 `auth-orm-plus` → `auth-orm-flex`、`dict-orm-plus` → `dict-orm-flex` 等），确保所有模块使用同一种 ORM 框架。同时需要将基础 ORM 依赖也做对应替换（如 `orm-mybatis` 替换为对应的基础依赖）。

## 如何扩展业务模块

本项目的设计理念是 **零业务代码，全依赖引入**。如需新增或移除业务模块，只需在 `pom.xml` 中添加或移除对应的依赖即可，无需编写任何代码。

每个业务模块通常包含三个依赖：

```xml
<!-- xxx 模块 -->
<dependency>
    <groupId>com.mcst</groupId>
    <artifactId>xxx-prd-bms</artifactId>     <!-- 接口定义 -->
</dependency>
<dependency>
    <groupId>com.mcst</groupId>
    <artifactId>xxx-server</artifactId>       <!-- 业务实现 -->
</dependency>
<dependency>
    <groupId>com.mcst</groupId>
    <artifactId>xxx-orm-plus</artifactId>     <!-- 数据持久化（根据所选 ORM 替换后缀） -->
</dependency>
```

框架会通过 `@ComponentScan("com.mcst")` 自动扫描并加载所有引入模块的组件。
